# Reflexión — TP8 Clon Móvil de Instagram

Usamos la IA (Claude) nás que nada como una ayuda para hacer preguntas puntuales, no para que nos tirara el proyecto entero de una. Al principio veníamos del TP anterior de Instagram Web en React, así que lo primero que le preguntamos fue cómo se traducía esa lógica a React Native con Expo, qué cambiaba, por qué no puedo usar `<div>` ni CSS normal, y cómo se arma la navegación con React Navigation. 
La consigna prohíbe usar `.map()` para renderizar la lista, así que le pedimos específicamente que lo hiciera con `FlatList` y nos explicara por qué es más eficiente (que solo renderiza lo que se ve en pantalla, no las 15 fotos juntas). También nos armó el patrón de `useEffect` + `axios.get` a The Cat API con los estados de `cargando` y `error`. Cada vez que algo no nos cerraba, le pegabamos el error o la línea concreta y le pedíamos que nos la explicara, en vez de copiar y pegar sin entender.
Hubo varias cosas que la IA nos dio de una forma y nosotros decidimos cambiarlas. La más importante es que la API de gatos solo devuelve el id y la url de la imagen, sin usuario, ni likes, ni comentarios. La IA nos hacía mostrar solo la foto, pero eso no se parecía a Instagram. Entonces creamos la función `construirPost()`, que mezcla la imagen real con datos simulados (autor, avatar, leyenda, ubicación, cantidad de likes y comentarios) que puse en `userData.js`. 
Aprendimos que en React Native el maquetado es todo con Flexbox y `StyleSheet.create()`, y que `FlatList` realmente cambia el rendimiento cuando la lista crece. Sobre el uso de IA, lo que más me quedó es que sirve muchísimo para crear una base y explicarte conceptos, pero si le pedís todo junto te da código que anda pero que no controlás.
